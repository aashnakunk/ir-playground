# BM25

The keyword-ranking workhorse of modern search. A refinement of [[TF-IDF]]
that handles long and short docs more fairly and saturates the reward for
repeated terms.

## Intuition

Same "count rare words" idea as TF-IDF, but with two practical fixes:

1. **Term-frequency saturation.** The 50th "pizza" in a doc helps barely more
   than the 5th. (TF-IDF would over-reward it.)
2. **Length normalization.** A 1-word text "pizza!" shouldn't automatically
   outrank a 500-word essay about pizza.

## How it works

For each query term `t` and doc `d`:

$$\text{score}(t, d) = \text{IDF}(t) \cdot \frac{\text{tf}(t,d) \cdot (k_1+1)}{\text{tf}(t,d) + k_1 \cdot (1 - b + b \cdot |d| / \text{avgdl})}$$

Doc's total score = sum over query terms.

- **k₁** controls TF saturation. Higher = more reward for repeated terms.
  Default ~1.5.
- **b** controls length normalization. 0 = ignore length, 1 = full
  normalization. Default ~0.75.

## When to use

- Default keyword scoring everywhere. Elasticsearch, Solr, OpenSearch,
  Lucene — all BM25 by default.
- The keyword side of [[Hybrid Search]].

## When not to use

- Pure semantic queries with no shared vocabulary. Use [[Semantic Search]] or
  combine via [[Hybrid Search]].

## Where it shows up

- Google for many years (now augmented with neural rerankers).
- Every modern RAG stack as the cheap retriever before [[Rerank]].
- The "keyword" half of [[Hybrid Search]].
