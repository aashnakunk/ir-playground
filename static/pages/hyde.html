HyDE

Let the AI write a draft answer first, then search using that draft. Counter-intuitively, this often finds better docs than searching the original question — even if the draft is wrong.

try this

  1. Ask a terse question (the kind a real user might type).
  2. Left: baseline — what you'd get by searching with the raw question.
  3. Middle: the hypothetical answer the LLM invented. It might be partially wrong — that's OK, we're not using it as truth, just as a richer query.
  4. Right: HyDE results — what you get by embedding that hypothetical answer instead.
  5. For terse questions, HyDE often surfaces a better top-1. For already-rich queries, plain semantic wins.
Think of it like: Googling. If you search "why does X happen" you get junk. If you search the answer you'd expect ("X happens because…") you get the right pages. HyDE asks the LLM to write that fake answer for you first.
why does this work?

Question embeddings and answer embeddings live in slightly different regions of the vector space. A short question doesn't look much like the long, factual passages in the corpus. The hypothetical answer does — even if it's hallucinated. So embedding-distance does a better job of finding the right passage.
