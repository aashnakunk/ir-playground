MCTS

Try random sequences of decisions, remember which ones paid off, then bias future tries toward those. The algorithm behind AlphaGo — and increasingly behind LLM agents that plan multi-step actions.

try this

  1. The tree has 3 root actions, each branching out 3-deep. Hidden rewards are at the leaves.
  2. Click run, then step through iterations. Each iteration does select (descend by UCB) → rollout (random play to a leaf) → backprop (update visit counts and value sums).
  3. Watch the per-node visit count and average value evolve. Nodes that get more visits are the ones the algorithm is converging on.
  4. The "root action summary" at the bottom shows the best root choice so far.
Think of it like: deciding which restaurant to try. You can't know how good each is without going — so you randomly try meals, remember which restaurants kept being good, and over time go back to those more often. UCB is the rule: "prefer good restaurants, but also occasionally check ones you haven't visited much yet."
where this shows up in modern AI

AlphaGo, AlphaZero, MuZero. Modern LLM-agent papers like LATS (Language Agent Tree Search) and Tree of Thoughts with value heads use MCTS variants to plan over candidate next actions. Powerful when individual rollouts are cheap and you have a value/reward signal.
