BFS / DFS

Two ways an AI agent explores a tree of possible next steps. Same graph; different order.

home

try this

  1. Click BFS or DFS to choose the algorithm, then run.
  2. Watch the frontier panel. BFS uses a queue (first in, first out — explore level-by-level). DFS uses a stack (last in, first out — dive deep).
  3. Step through with prev/next. Notice BFS finds the shortest-path goal first; DFS might wander deep on one branch before backing up.
  4. Click any log row to jump to that step.
Think of it like: looking for your keys. BFS = check every spot 1 foot away first, then 2 feet, then 3. DFS = pick a corner, search it top to bottom, then move to the next corner.
where this shows up in modern AI

Tree of Thoughts reasoning explores candidate "next steps" as a search tree — BFS-style ToT keeps multiple thoughts alive per level, DFS-style ToT commits to one branch and backtracks if it fails. ReAct agents do depth-first action chains.
