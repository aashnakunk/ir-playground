CAG

Skip the lookup step entirely. Hand the AI your entire corpus upfront and let it find what's relevant on its own. Modern LLMs auto-cache the corpus, so the 2nd question onwards is near-instant.

  4. For context: a typical "look it up then answer" pipeline sends ~250 prompt tokens. This sends ~1,800 every time. Without auto-caching, CAG would be 7× more expensive — caching is the whole reason it's viable.
  5. Ask the tutor: "when would RAG be better than CAG?"
Think of it like: memorizing the textbook before the exam vs. flipping to a chapter mid-answer (that's RAG). CAG is slower per question if you do it once, but caching makes the second question onwards basically free.
why use CAG at all?

RAG can fail when the best chunk wasn't retrieved (or the question needs context spread across many chunks). CAG sidesteps that by always giving the model everything. The tradeoff is cost and context window: CAG only works if your corpus fits in the model's context, and only stays cheap if you reuse the same prompt enough to amortize caching.
