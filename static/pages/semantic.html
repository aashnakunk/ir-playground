try this

  1. Type a query in full sentences, the way you'd ask a friend. The model understands meaning, not just keywords.
  2. The ★ on the map is your query. The closest dots are the most similar docs by meaning. Top-k are ringed in orange.
  3. Try a query that uses none of the exact words from any doc. Notice it still finds the right one — that's the magic.
  4. Hover any dot on the map to see its title and similarity score.
  5. Try a query with none of the exact words from any doc. It'll still find the right one — that's the headline trick. Keyword search would have returned nothing.
Think of it like: Shazam for ideas. You hum the meaning; it finds docs that hum a similar tune — even if they use totally different words.
what's actually happening?

Each doc and your query are turned into a list of 1,536 numbers (an "embedding") by an OpenAI model. Similar meanings produce similar number-lists. Ranking = which list is most similar to your query's list, measured by cosine similarity. The 2D map is a flattened projection so you can see clusters.
