How does search actually work?

A hands-on tour of the seven techniques behind every search box and chatbot.

boolean tfidf bm25 semantic hybrid mmr hnsw rerank hyde rag cag

When you type into a search box — Google, Spotify, ChatGPT — something has to decide which results to show. Sometimes it counts words. Sometimes it compares meanings. Sometimes it draws a graph. This site lets you watch each of those techniques run on a small example corpus, and ask a tutor questions about what you're seeing.

No background needed. Each page has a "Try this" guide. The sidebar on the right is a tutor (powered by OpenAI) that can see your screen — ask it anything.

recommended path

Each technique builds on the one before it. Go in order if it's your first time.

  1. TF-IDF The original "count the words" search. A doc wins if it has lots of the rare words from your query.
  2. BM25 Same idea, smarter math. Doesn't get fooled by very long or very short docs. Google used this for years.
  3. Semantic search Search by meaning, not exact words. Type "electric music in the 60s" and it finds the doc about jazz fusion even if those words aren't in it.
  4. Hybrid search Combine keyword search and meaning search. Usually beats either one alone.
  5. HNSW The trick that makes semantic search fast when you have millions of docs. A layered graph you can hop across.
  6. RAG "Look it up first, then answer." How a chatbot grounds its answers in real docs instead of making things up.
  7. CAG Skip the lookup — hand the chatbot the whole corpus and let it figure out what's relevant. Simpler, but costs more tokens.

going deeper

Variants and tricks layered on top of the basics. Optional, but useful once you've done the recommended path.

  1. Boolean search + inverted index Pre-search-engine search. Type a AND NOT b; watch the engine intersect sets of doc-ids from the underlying index.
  2. MMR (diversity re-ranking) If your top-5 results are all near-duplicates, MMR re-orders them to cover more ground. λ slider lets you choose relevance vs diversity.
  3. Re-ranking (2-stage retrieval) Retrieve cheap, then re-score with a heavier model. The standard production-RAG recipe.
  4. HyDE (LLM-assisted retrieval) Ask the LLM to invent a hypothetical answer first, then embed THAT to search. Often beats embedding a short question directly.

how to use this site

  • Type queries and watch the visuals update — that's the whole point.
  • Hover the dotted underlined words for short definitions (try "IDF", "k1", "efSearch").
  • Talk to the tutor in the sidebar — it can see what's on your screen. Ask "why did doc #7 win?" or "what would change if I increased M?"
  • Compare pages. Run the same query through TF-IDF, BM25, and Semantic and notice how the rankings differ.
