Core retrieval used in production today

Given a query and a pile of docs, return the most relevant ones. The bread-and-butter of every RAG system.

  1. BM25 The keyword-matching workhorse. Used by Elasticsearch, Lucene, and basically every search backend.
  2. Semantic search Cosine similarity over embeddings. Finds docs by meaning, not by exact words.
  3. Hybrid search Combine BM25 + semantic. Usually beats either alone — it's the modern RAG default.
  4. HNSW The data structure that makes semantic search fast at million-doc scale. Step-mode shows the math.

RAG pipelines & enhancements used in production today

Once you can find relevant docs, what do you do with them? These techniques produce the answer.

  1. RAG Retrieve top-k, paste into the prompt, generate. The standard recipe.
  2. CAG Skip retrieval — hand the model the entire corpus, cached. Cheaper than it sounds for repeat queries.
  3. MMR Diversity re-ranking. Stops your top-5 from being five near-duplicates.
  4. Re-ranking Cheap retriever first, smarter model second. The 2-stage trick behind almost every production RAG.
  5. HyDE LLM writes a hypothetical answer first; we search with that. Often beats embedding a terse question.

Agent search algorithms used in modern reasoning agents

These don't find docs — they find paths, sequences, or plans. Powers Tree-of-Thoughts reasoning, LLM token generation, and tree-search-based planning agents.

  1. BFS / DFS Explore a tree of next reasoning steps level-by-level (BFS) or depth-first (DFS). The skeleton of Tree-of-Thoughts.
  2. A* BFS plus a "how close to the goal" estimate. The classic pathfinding algorithm; used in some agent planners.
  3. Beam search How LLMs pick their next word. Keep the top-k partial sequences, expand, prune to top-k again.
  4. MCTS Random rollouts + UCB. Behind AlphaGo and modern LLM-agent papers like LATS.

Foundational history & background

The algorithms BM25 and TF-IDF improved on. Not what you'd reach for first today, but understanding them makes everything above click.

  1. Boolean + inverted index Pre-1970s keyword search. Type a AND NOT b; see the underlying data structure every modern keyword engine still uses.
  2. TF-IDF "Count the rare words." The first scoring scheme. BM25 is a direct refinement.
