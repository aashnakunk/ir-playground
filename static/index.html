How does search actually work?

github

When you type into a search box — Google, Spotify, YouTube, ChatGPT — something has to decide which results to show. Sometimes it counts words. Sometimes it compares meanings. Sometimes it draws a graph and walks across it.

This site lets you watch each of those techniques run on a small example corpus. Pick a tile below. Every page has a "try this" guide and a sidebar tutor that can see what's on your screen — ask it anything.

Core retrieval

in production today
BM25The keyword-matching workhorse. Used by Elasticsearch, Lucene, every search backend. Semantic searchFind docs by meaning, not exact words. Powered by embeddings. Hybrid searchCombine keyword + meaning. The modern RAG default. HNSWThe graph that makes semantic search fast at million-doc scale.

RAG pipelines

in production today
RAGLook it up, then answer. How a chatbot grounds replies in real docs. CAGSkip lookup. Cache the whole corpus in the prompt. MMRDiversity re-ranking. Stops your top-5 from being near-duplicates. RerankCheap retriever first, smarter model second. The 2-stage trick. HyDELLM writes a hypothetical answer; we search with that.

Agent search
